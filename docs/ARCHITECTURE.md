### 内存布局

```
高地址
┌──────────────────────────┐
│   模块代码区 (16KB)       │  ← 非易失性
│   module_code[]          │
├──────────────────────────┤
│   堆 (8KB)               │  ← 非易失性
│   heap[]                 │
│   ↑ 向上增长              │
├──────────────────────────┤
│   全局数据区 (4KB)        │  ← 易失性
│   global_data[]          │
├──────────────────────────┤
│   间接变量栈 (64×16B)     │  ← 易失性
│   indirect_var_stack[][] │
│   ↓ 向下增长              │
├──────────────────────────┤
│   执行器栈 (256×4B)       │  ← 易失性
│   executor_stack[]       │
│   ↓ 向下增长              │
└──────────────────────────┘
低地址
```

### 指令执行流程

```
程序计数器 (PC)
   │
   ▼
┌─────────────┐
│   取指       │ ◄── 从module_code[PC]读取操作码
└─────────────┘
   │
   ▼
┌─────────────┐
│   解码       │ ◄── 解析操作数和指令长度
└─────────────┘
   │
   ▼
┌─────────────┐
│   执行       │ ◄── 根据opcode调用相应处理函数
└─────────────┘
   │
   ├──► 算术指令 ──► 从栈弹出操作数，计算，结果压栈
   ├──► 逻辑指令 ──► 位运算，结果压栈
   ├──► 比较指令 ──► 比较，布尔值压栈
   ├──► 跳转指令 ──► 修改PC
   ├──► 调用指令 ──► 创建栈帧，跳转
   ├──► 返回指令 ──► 销毁栈帧，恢复PC
   └──► 内存指令 ──► 读写内存
   │
   ▼
┌─────────────┐
│  更新PC      │ ◄── PC += 指令长度
└─────────────┘
   │
   └──► 回到取指阶段
```

### 函数调用机制

```
调用者                          被调用函数
   │                                 │
   │  1. 参数压栈                     │
   │  ───────────────────────────►   │
   │                                 │
   │  2. CALL指令                    │
   │  ───────────────────────────►   │
   │                                 │  3. 创建栈帧
   │                                 │     push_frame()
   │                                 │     ├─ 保存返回地址
   │                                 │     ├─ 保存BP
   │                                 │     └─ 分配局部变量
   │                                 │
   │                                 │  4. 执行函数体
   │                                 │     ├─ 加载局部变量
   │                                 │     ├─ 执行计算
   │                                 │     └─ 存储结果
   │                                 │
   │                                 │  5. RET指令
   │  ◄───────────────────────────   │
   │                                 │  6. 销毁栈帧
   │                                     pop_frame()
   │                                     ├─ 恢复BP
   │                                     ├─ 恢复PC
   │                                     └─ 返回值压栈
   │
   │  7. 从栈获取返回值
   ▼
```

### 事务管理机制

```
应用请求
   │
   ▼
┌─────────────────┐
│ transaction_    │
│   begin()       │
└─────────────────┘
   │
   ├─► 备份堆数据
   ├─► 备份全局数据
   └─► 设置active=true
   │
   ▼
┌─────────────────┐
│   执行操作       │
│  (可能多次)      │
└─────────────────┘
   │
   ├─► 成功 ──► commit()
   │             ├─► 释放备份
   │             └─► active=false
   │
   └─► 失败 ──► rollback()
               ├─► 恢复堆数据
               ├─► 恢复全局数据
               ├─► 释放备份
               └─► active=false
```

## 关键数据结构

### VMContext (虚拟机上下文)

```
VMContext
├─ 运行时数据区
│  ├─ executor_stack[256]        # 执行器栈
│  ├─ stack_pointer              # 栈指针
│  ├─ base_pointer               # 基址指针
│  ├─ frame_stack[64]            # 栈帧栈
│  ├─ frame_top                  # 栈帧栈顶
│  ├─ indirect_var_stack[64][16] # 间接变量栈
│  ├─ indirect_stack_pointer     # 间接栈指针
│  ├─ global_data[4096]          # 全局数据区
│  ├─ heap[8192]                 # 堆
│  ├─ heap_used                  # 堆使用量
│  ├─ program_counter            # 程序计数器
│  └─ module_code[16384]         # 模块代码区
│
├─ 模块管理
│  ├─ modules[32]                # 模块数组
│  └─ module_count               # 模块数量
│
├─ 应用管理
│  ├─ apps[512]                  # 应用实例数组
│  └─ app_count                  # 应用数量
│
├─ 当前状态
│  ├─ current_module             # 当前模块
│  ├─ current_app                # 当前应用
│  ├─ current_channel            # 当前通道
│  └─ state                      # VM状态
│
├─ 异常处理
│  ├─ exception                  # 当前异常
│  ├─ exception_handlers[16]     # 异常处理器
│  └─ handler_count              # 处理器数量
│
├─ 事务管理
│  └─ transaction                # 事务上下文
│     ├─ active                  # 是否激活
│     ├─ backup_data             # 备份数据
│     ├─ backup_size             # 备份大小
│     └─ checkpoint_count        # 检查点数
│
└─ 统计信息
   ├─ instructions_executed      # 已执行指令数
   └─ total_execution_time       # 总执行时间
```

## 指令编码示例

### LDC.U8 (加载8位无符号常数)

```
格式: [OPCODE] [IMM8]
示例: LDC.U8 42

字节码: 0x4A 0x2A
         │    │
         │    └─ 立即数 42
         └────── 操作码 OP_LDC_U8

执行:
  栈前: []
  栈后: [42]
```

### ADD (加法)

```
格式: [OPCODE]
示例: ADD

字节码: 0x60
         │
         └─ 操作码 OP_ADD

执行:
  栈前: [5] [3]
  栈后: [8]
  
  val2 = pop()  # 3
  val1 = pop()  # 5
  result = val1 + val2  # 8
  push(result)
```

### BEQZ.S8 (为零则跳转)

```
格式: [OPCODE] [OFFSET8]
示例: BEQZ.S8 -10

字节码: 0x12 0xF6
         │    │
         │    └─ 偏移 -10 (补码)
         └────── 操作码 OP_BEQZ_S8

执行:
  val = pop()
  if val == 0:
      PC += -10
```
